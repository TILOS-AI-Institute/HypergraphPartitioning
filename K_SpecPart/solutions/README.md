# K-SpecPart solutions

Best K-SpecPart partitions on the Titan23 suite, produced by the benchmark
sweep ([`../titan_sweep.jl`](../titan_sweep.jl)) starting from the hMETIS hints
in [`../hints/`](../hints/) at imbalance `ub_factor = 2`.

For each `(design, K)`, this folder holds the **lowest-cut solution over the 3
seeds** — i.e. the "K-SpecPart best" column of the tables in the top-level
[`README`](../README.md#benchmarks). The exact cut sizes, the winning seed, and
per-seed statistics are in `titan_sweep_results/best_cuts.csv` (regenerated by
`collect_best_cuts.jl`).

## Layout

```
solutions/
└── Titan23/
    └── ub_factor_2/
        ├── 2_way/  <design>.hgr.kspecpart.ubfactor.2.part.2
        ├── 3_way/  <design>.hgr.kspecpart.ubfactor.2.part.3
        └── 4_way/  <design>.hgr.kspecpart.ubfactor.2.part.4
```

22 designs × K ∈ {2, 3, 4} = 66 partitions.

## Format

hMETIS-style partition: **one 0-indexed block id per line**, in vertex order
(the same format as the hint files and `run_kspecpart.jl --out`).

> These are the K-SpecPart *outputs*; the corresponding *inputs* (hMETIS hints)
> are under `../hints/`. Both are tracked as explicit exceptions to the
> `*.part.*` rule in the top-level `.gitignore`.
